# Login/Logout Tracking System Documentation

## Overview
This document explains how the login and logout tracking system works in the Online Exam Management System.

## System Flow

### 1. Login Process

#### Step 1: User Enters Credentials
- User enters username and password on the Login page (`/src/pages/Login.tsx`)
- Clicks "Sign In" button

#### Step 2: Authentication
- `signIn()` function is called from `useAuth` hook (`/src/hooks/useAuth.ts`)
- Supabase authenticates the user credentials
- If authentication fails, error message is shown
- If authentication succeeds, proceed to Step 3

#### Step 3: Login Tracking (Immediate)
**Location**: `/src/hooks/useAuth.ts` - `signIn()` function (lines 49-91)

After successful authentication, the system **immediately**:

1. **Fetches user profile** from the database
2. **Creates login history record** by calling `loginHistoryApi.createLoginHistory()`:
   - `user_id`: User's unique identifier
   - `username`: User's username
   - `full_name`: User's full name
   - `role`: User's role (admin/principal/teacher/student)
   - `school_id`: Associated school ID
   - `login_time`: Current timestamp (auto-generated by database)
   - `ip_address`: Currently NULL (would need backend to capture real IP)
   - `user_agent`: Browser information (e.g., "Mozilla/5.0...")
   - `logout_time`: NULL (will be updated on logout)

3. **Creates/updates active session** by calling `activeSessionApi.upsertActiveSession()`:
   - Tracks currently active users
   - Updates last activity timestamp

#### Step 4: Success Message
- Toast notification shows: **"Login successful"**
- User is redirected to their role-based dashboard

### 2. Logout Process

#### Step 1: User Clicks Logout
- User clicks "Logout" from the dropdown menu in Header (`/src/components/common/Header.tsx`)
- `handleSignOut()` function is called

#### Step 2: Logout Tracking (Immediate)
**Location**: `/src/hooks/useAuth.ts` - `signOut()` function (lines 123-136)

Before signing out, the system **immediately**:

1. **Updates logout time** by calling `loginHistoryApi.updateLogoutTime()`:
   - Finds the most recent login record for this user where `logout_time` is NULL
   - Updates the `logout_time` field with current timestamp

2. **Updates active session** by calling `activeSessionApi.logoutSession()`:
   - Changes session status to 'logged_out'
   - Records logout timestamp

#### Step 3: Sign Out
- Supabase signs out the user
- Session is cleared

#### Step 4: Success Message
- Toast notification shows: **"Logged out successfully"**
- User is redirected to the login page

## Database Tables

### login_history Table
```sql
CREATE TABLE login_history (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  username TEXT NOT NULL,
  full_name TEXT,
  role TEXT NOT NULL,
  school_id UUID REFERENCES schools(id) ON DELETE SET NULL,
  login_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  logout_time TIMESTAMPTZ NULL,  -- Updated on logout
  ip_address TEXT,
  user_agent TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### active_sessions Table
```sql
CREATE TABLE active_sessions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  username TEXT NOT NULL,
  full_name TEXT,
  role TEXT NOT NULL,
  school_id UUID REFERENCES schools(id) ON DELETE SET NULL,
  login_time TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  last_activity TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  ip_address TEXT,
  user_agent TEXT,
  status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'idle', 'logged_out')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(user_id)
);
```

## API Functions

### Login History API (`/src/db/api.ts`)

#### createLoginHistory()
```typescript
async createLoginHistory(
  userId: string,
  username: string,
  fullName: string | null,
  role: string,
  schoolId: string | null,
  ipAddress: string | null = null,
  userAgent: string | null = null
): Promise<LoginHistory | null>
```
- **Purpose**: Creates a new login history record
- **Called**: Immediately after successful authentication
- **Returns**: The created login history record

#### updateLogoutTime()
```typescript
async updateLogoutTime(userId: string): Promise<void>
```
- **Purpose**: Updates the logout_time for the most recent login record
- **Called**: Immediately before signing out
- **Logic**:
  1. Finds the most recent login record where `logout_time` is NULL
  2. Updates `logout_time` with current timestamp

### Active Session API (`/src/db/api.ts`)

#### upsertActiveSession()
```typescript
async upsertActiveSession(
  userId: string,
  username: string,
  fullName: string | null,
  role: string,
  schoolId: string | null,
  ipAddress: string | null = null,
  userAgent: string | null = null
): Promise<ActiveSession | null>
```
- **Purpose**: Creates or updates active session record
- **Called**: Immediately after successful authentication
- **Behavior**: Uses UPSERT to update existing session or create new one

#### logoutSession()
```typescript
async logoutSession(userId: string): Promise<void>
```
- **Purpose**: Marks session as logged out
- **Called**: Immediately before signing out
- **Updates**: Sets status to 'logged_out' and records logout timestamp

## Key Features

### 1. Immediate Tracking
- **No delays**: Login and logout details are captured immediately
- **No background jobs**: All tracking happens synchronously during login/logout
- **No duplicate prevention**: Every login creates a new record (removed `hasLoggedInToday()` check)

### 2. Error Handling
- Login/logout tracking errors are logged but **do not block** authentication
- Users can still login/logout even if tracking fails
- Errors are logged to console for debugging

### 3. User Feedback
- **Login Success**: "Login successful" toast message
- **Logout Success**: "Logged out successfully" toast message
- **Login Failure**: Error message with details
- **Logout Failure**: Error message with details

### 4. Data Captured

#### On Login:
- User ID
- Username
- Full Name
- Role (admin/principal/teacher/student)
- School ID
- Login Timestamp (auto-generated)
- Browser Information (User Agent)
- IP Address (currently NULL, needs backend implementation)

#### On Logout:
- Logout Timestamp (updates existing record)

## Technical Notes

### 1. IP Address Capture
Currently, `ip_address` is set to NULL because:
- Client-side JavaScript cannot reliably get the real IP address
- Would require backend/edge function to capture server-side IP
- Can be implemented using Supabase Edge Functions if needed

### 2. Browser Information
- Captured using `navigator.userAgent`
- Provides browser type, version, and operating system
- Example: "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36..."

### 3. Session Management
- Each user can have only one active session (enforced by UNIQUE constraint on user_id)
- New login updates the existing active session
- Logout marks session as 'logged_out'

### 4. Inferred Logout Times
The system includes a function to infer logout times for records where users didn't explicitly logout:
- If a user logs in again, previous session's logout time is set to new login time
- If no subsequent login after 24 hours, logout time is set to login_time + 8 hours

## Testing the System

### Test Login Tracking:
1. Open browser developer console (F12)
2. Login with valid credentials
3. Check console for "Error tracking login" (should not appear)
4. Verify toast message: "Login successful"
5. Check database: `SELECT * FROM login_history ORDER BY login_time DESC LIMIT 1;`
6. Verify all fields are populated (except logout_time should be NULL)

### Test Logout Tracking:
1. While logged in, open browser developer console
2. Click "Logout" from dropdown menu
3. Check console for "Error updating session status" (should not appear)
4. Verify toast message: "Logged out successfully"
5. Check database: `SELECT * FROM login_history WHERE user_id = 'YOUR_USER_ID' ORDER BY login_time DESC LIMIT 1;`
6. Verify `logout_time` is now populated with timestamp

## Troubleshooting

### Login not being tracked:
1. Check browser console for errors
2. Verify Supabase connection is working
3. Check RLS policies on `login_history` table
4. Verify user profile exists in `profiles` table

### Logout time not updating:
1. Check browser console for errors
2. Verify the login record exists with NULL logout_time
3. Check RLS policies allow UPDATE on `login_history` table
4. Verify user_id matches the logged-in user

### Missing data in records:
1. Check if profile data is complete in `profiles` table
2. Verify school_id exists if user is associated with a school
3. Check browser compatibility for `navigator.userAgent`
